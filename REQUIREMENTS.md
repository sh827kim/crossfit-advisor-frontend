# 애프터와드 (Afterword) Frontend

Crossfit **초심자**를 위한
보강운동 추천 및 운동 기록 관리를 제공하는 **프론트엔드 애플리케이션**이다.

사용자는 3가지 운동 모드를 선택하여:
1. **부족한 부위 채우기**: 오늘 WOD에서 안 쓴 근육 위주의 보강운동
2. **나의 달성 목표**: 특정 스킬(머슬업, 핸드스탠드 등) 연습
3. **타겟 부위 선택**: 원하는 부위 집중 공략

원하는 운동 시간을 선택하면 맞춤형 운동 계획이 생성되고,
실시간 타이머와 함께 운동을 진행하며
완료 후 기록을 저장할 수 있다.

본 문서는 **현재 구현된 기능 기준**이며,
향후 기능 확장을 고려한 **확장 가능한 설계**를 전제로 한다.

---

## 1. Project Goal

- 운동 모드 선택 → 운동 계획 생성 → 타이머와 함께 실시간 진행 → 기록 저장
- Mobile 환경에서 언제든 접근 가능한 직관적 UI 제공
- 간편한 운동 추적 및 기록 관리 시스템 구현

---

## 2. Target Platform & Constraints

### Platform
- Mobile Web 중심
- **PWA 형태** (설치 가능, 기본 캐싱)
- Backend API 연동 구조

### 인증
- 현재 MVP에서는 인증 제거 (로컬 테스트 환경)
- 향후 Google OAuth 기반 로그인 추가 예정

---

## 3. 현재 구현된 기능

### 3.1 홈 페이지 (메인 메뉴)

3가지 운동 모드 선택 버튼:

#### 1) 부족한 부위 채우기 (WOD 모드)
- 사용자가 수행한 WOD 운동 입력
- 입력한 운동의 근육그룹 분석
- 사용하지 않은 근육 위주 보강운동 추천
- 백엔드: `/api/v1/workouts/generate/wod`

#### 2) 나의 달성 목표 (GOAL 모드)
- 연습 목표 선택 (머슬업, 핸드스탠드 등)
- 해당 목표 달성을 위한 운동 추천
- 백엔드: `/api/v1/workouts/generate/goal`

#### 3) 타겟 부위 선택 (PART 모드)
- 집중할 부위 선택 (CHEST, BACK, LEGS, ARMS, SHOULDERS, CORE)
- 선택 부위 집중 공략 운동 추천
- 백엔드: `/api/v1/workouts/generate/part`

---

### 3.2 운동 시간 선택

- 선택 가능 시간: 1, 5, 10, 15, 20, 25, 30분
- 시간에 따라 운동 개수 및 세트 자동 결정

---

### 3.3 운동 계획 생성

백엔드 API 응답:
- 운동 목록 (운동명, 장비, 권장 횟수)
- 라운드 수
- 라운드당 목표 시간

---

### 3.4 실시간 운동 진행 (Result 페이지)

- **타이머 표시** (MM:SS 형식)
- **각 운동별 완료 체크박스**
  - 체크 시 녹색으로 표시, 운동명 취소선 처리
- **운동별 횟수 조절** (최소-최대 범위)
  - 수행한 실제 횟수 기록 가능
- **완료 진행률 표시** (X/Y)

사용자 상호작용:
- 운동 시작/일시정지 버튼으로 타이머 제어
- 각 운동 완료 시 체크
- 횟수 범위 내에서 실제 수행 횟수 기록
- 모든 운동 완료 시 기록 저장 가능

**"기록 없이 돌아가기" 버튼**:
- 운동 미완료 시에도 항상 표시 (운동 중단 가능)
- 클릭 시 확인 팝업 표시
- "정말 그만하시겠어요?" 메시지
- 확인 시에만 홈으로 돌아가기

---

### 3.5 운동 기록 저장

모든 운동 완료 후:
- "오늘 운동 기록하기" 버튼으로 저장
- 저장 항목:
  - 운동 일자 (YYYY-MM-DD)
  - 운동 모드 (WOD / GOAL / PART)
  - 소요 시간 (분)
  - 각 운동별 수행 현황 (운동명 + 횟수)

기록은 **localStorage**에 저장

---

### 3.6 운동 기록 관리

#### 히스토리 페이지
- 월별 캘린더 뷰 (기록 있는 날짜 파란색으로 강조)
- 선택 날짜의 운동 기록 상세 조회
- **기본으로 오늘 날짜 기록 표시**
- 월 이동 가능

#### 프로필 페이지
- **닉네임 편집**
  - 편집 버튼 클릭 시 입력 필드 표시
  - 저장/취소 버튼
  - localStorage에 지속 저장
- **현재 월 운동 횟수 표시**

---

## 4. UX / UI Guidelines

- Mobile First
- 한 손 조작 중심 UI
- 명확한 로딩 상태 표시
- 버튼 간격으로 명확한 구분
- 운동 진행 중 의도치 않은 나가기 방지 (확인 팝업)

---

## 5. 아키텍처 & 상태 관리

### Context API 사용
- 전역 상태: `AppContext`
  - 현재 운동 모드
  - 선택된 운동 목록 (WOD, GOAL, PART)
  - 생성된 운동 계획
  - 운동 기록 (localStorage 동기화)
  - 총 운동 시간

### 데이터 저장
- **운동 계획**: 메모리 (Context)
- **운동 기록**: localStorage (date별 저장)
- **사용자 정보**: localStorage (닉네임)

---

## 6. 향후 확장 기능

- **AI 대화 기능**: 음성 입력 및 음성 피드백
- **운동 강도 조절**: 난이도 레벨 선택 (초급 / 중급 / 고급)
- **통계 & 시각화**
  - 주간 / 월간 운동량 그래프
  - 운동 종목별 기록 추이
- **개인 기록 (PR) 관리**
  - 종목별 최고 기록
  - 기록 날짜 추이
- **클라우드 동기화**
  - 백엔드 계정 연동
  - 다기기 동기화
- **PWA 최적화**
  - Push 알림 (운동 리마인더)
  - 오프라인 모드 지원

---

## 7. Non-goals (하지 않는 것)

- 커뮤니티 기능
- 소셜 공유
- 운동 영상 가이드
- 전문 코치 상담
- 구독/결제 기능

---

## 8. Guiding Principle

> 애프터와드의 목표는
> **"운동 모드 선택 → 계획 생성 → 실시간 추적 → 기록 저장" 흐름을 제공하는 것**이다.
>
> 모든 구조는 이후 통계·AI 기능이 자연스럽게 확장될 수 있도록 설계한다.
> 사용자는 복잡한 설정 없이 빠르게 운동을 시작하고 기록할 수 있어야 한다.
